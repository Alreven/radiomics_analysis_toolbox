# Radiomics analysis scripts

This repository contains Python scripts used to extract radiomics features from MRI/DICOM images and to perform reproducibility/statistical analyses.

## Environment

The code was prepared for **Python 3.9**.

Install the required packages with:

```bash
python3.9 -m venv .venv
source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

## Scripts

- `irmtech.py`: extracts PyRadiomics features from DICOM folders using NIfTI masks and writes the results to `radiomics_features.csv`.
- `irmtech_lib.py`: helper functions for image preprocessing, including min-max normalization, fixed bin number normalization, and N4ITK bias field correction with cache.
- `mask_maker.py`: interactive tool for creating a NIfTI mask from a DICOM series; exports `mask_volume.nii`.
- `icc_calculation.py`: calculates ICC(2,1) values from `radiomics_features.csv` and writes `icc_results.csv`.
- `cov_calculation.py`: calculates coefficient of variation values from `radiomics_features.csv` and writes `cv_results.csv`.
- `wilcoxon_unilateral_icc.py`: performs a paired one-sided Wilcoxon test comparing ICC values between two preprocessing conditions.
- `wilcoxon_unilateral_cv.py`: performs a paired one-sided Wilcoxon test comparing CV values between two preprocessing conditions.
- `fdr_correction.py`: applies Benjamini-Hochberg FDR correction to a list of p-values.

## Expected input structure

By default, `irmtech.py` expects:

```text
src/data/   # DICOM folders
src/masks/  # NIfTI masks (.nii)
```

The output feature table is written to:

```text
radiomics_features.csv
```

Some scripts use hard-coded input filenames and preprocessing conditions. Edit the variables at the top of each script before running them if your filenames or conditions differ.

## Example usage

```bash
python irmtech.py
python icc_calculation.py
python cov_calculation.py
python wilcoxon_unilateral_icc.py
python wilcoxon_unilateral_cv.py
python fdr_correction.py
```

## Notes

The repository contains code only. Imaging data, masks, and generated CSV files should be provided separately or generated by the scripts.
